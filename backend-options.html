📊 Google Sheets via Apps Script

FREE ⭐ Recommended Free

Game POSTs JSON to a Google Apps Script web app, which writes rows to a Google Sheet.

🎮BrowserOKR Miner Game
⚙️Apps ScriptdoPost() handler
📊Google SheetRows of data
👁️YouView in Sheets
Setup Time~15 minutes
Cost$0 forever
Request Limits20,000 requests/day
Latency~2 seconds
Query CapabilitySheets formulas / QUERY()
Real-time?No
DashboardGoogle Sheets (built-in charts)

✓ Pros

  • Zero cost — no credit card, no subscription
  • No server to manage or deploy
  • Familiar UI — everyone knows Google Sheets
  • Easy to share with teammates
  • Built-in charting and pivot tables

✗ Cons

  • 20,000 requests/day limit
  • ~2 second latency per write
  • Data lives in Google's cloud
  • No real-time dashboard

💻 Code Preview

game-side · JavaScript // In game — send analytics on session end fetch('https://script.google.com/macros/s/YOUR_ID/exec', { method: 'POST', mode: 'no-cors', body: JSON.stringify(analyticsData) });
server-side · Apps Script // Google Apps Script — doPost handler function doPost(e) { var data = JSON.parse(e.postData.contents); var sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet(); sheet.appendRow([ new Date(), data.duration, data.mode, data.milestones, data.peakOKR, data.buildingsPlaced ]); }

🔧 GitHub Issues as Database

FREE 🔧 Most Creative

Game creates a GitHub Issue via the API for each session, using issue labels for filtering — creative and completely free.

🎮BrowserOKR Miner Game
🐙GitHub APIREST v3
📋RepositoryIssues as data
🔍SearchGitHub Issue Search
Setup Time~10 minutes
Cost$0 forever
Request Limits5,000 requests/hr (authenticated)
Latency~500ms
Query CapabilityGitHub Issue search & labels
Real-time?No
DashboardGitHub Issues page + Actions

✓ Pros

  • Completely free — no limits on storage
  • Built-in search & filter with labels
  • Data lives alongside your code
  • GitHub Actions can process data automatically
  • Issue labels for easy categorization

✗ Cons

  • Not a real database — no SQL queries
  • Token exposed in client (use fine-grained with issue:write only)
  • Rate limited to 5,000 requests/hr
  • Messy for high volume — lots of issues
  • Visible to public on public repos
